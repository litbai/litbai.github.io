使用 Hexo+Github 搭建个人博客

工作后一直想写个人博客，但由于自控力不足，迟迟没有开写。至今，工作已1年有余，工作经验确虽然也在不断提升，但是会发现很多知识并没有沉淀下来，特别是对于业务开发人员来说。当需要某方面知识的时候，还是要去现搜使用姿势。比如，使用gitbook生成静态站点，当初花了2天功夫搞定了，但并没有做沉淀，现在如果想使用gitbook生成一个静态站点，还是要去搜具体的步骤。再比如，现在个人博客是我使用Hexo+Github来搭建的，其实在半年前我用Jekyll+Github搭建过个人博客，但由于没有沉淀，只记得当初搭建时踩了不少坑，所以现在才想要使用Hexo+Github从头搭建一个新的个人博客。

温故而知新

孔子《论语》

至此，一切从搭建个人博客开始，逐渐开始沉淀知识。本篇博客就记录了我自己使用Hexo+Github搭建个人博客的全过程。

一、安装Hexo

What is Hexo？

Hexo（https://hexo.io/） 是一个快速的，简单的，强大的博客框架。它的特点是：速度极快（Blazing Fase）、Markdown语法支持、一键部署（One-Commandm Deployment）、插件众多。说白了，就是你可以使用Markdown格式来写文章，然后Hexo就可以在几秒内为你生成具有精美主题的静态网页，当然它的功能不止如此。

二、关联hexo + github pages

What is Github Pages？

Github Pages提供静态网站托管服务，它本身不支持动态内容。它可以直接托管你的git工程，提供免费域名，也就是可以使你的静态网页，直接发布到公网。

Github官网推荐使用Jekyll博客框架，并且提供了如何整合Jekyll的官方教程。但是个人感觉Hexo更为简单，而且背后的整合道理是一致的，选取哪个博客框架，看个人喜好，并没有优劣之分。

以后每次部署的步骤：
  1. 写博客（写md文档）
  2. hexo server 本机起hexo服务，预览效果
  3. hexo clean 清除上一次生成的文件内容
  4. hexo generate 生成静态网页
  5. hexo deploy 部署到远端
  6. username.github.io 观察远端效果
