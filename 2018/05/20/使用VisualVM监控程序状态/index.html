使用VisualVM监控程序状态

Visual VM

Visual VM 是1.6版本之后， jdk自带的一个JVM性能监控工具，以可视化的方式向开发者展示JVM的各种运行时信息，包括CPU、内存、线程、GC等，方便开发者进行故障定位以及性能调优。

Visual VM具有如下功能：

  • 监控本地或远程java进程
  • 显示JVM参数配置及运行时环境配置（系统属性）
  • 动态监视内存、CPU、类、线程信息
  • 线程dump、内存dump，生成快照
  • 分析已有的head dump文件（如通过jmap -dump:format=b,file=xxx pid生成的hprof文件）
  • 性能分析，包括内存分配、方法调用时间和频率
  • 插件机制，包括常用的VisualGC、BTrace插件

使用Visual VM可以监控本地java进程以及远程java进程。

监控远程java进程

Visual VM也可以监控远程java进程，需要如下步骤：

  • Visual VM 配置远程主机信息：右击“远程”选项，选择“添加远程主机”，输入远程主机IP和显示名称

  • 启动Visual VM

  • 远程服务器运行jstatd命令： jstatd是一个监控 JVM 从创建到销毁过程中资源占用情况并提供远程监控接口的 RMI （ Remote Method Invocation ，远程方法调用）服务器程序，它是一个 Daemon 程序，要保证远程监控软件连接到本地的话需要 jstatd 始终保持运行。 jstatd运行需要通过 -J-Djava.security.policy=* 指定安全策略，因此我们需要在服务器上建立一个指定安全策略的文件jstatd.all.policy ，文件内容如下：
